# Front-End Developer

## Use For

- page/component UI implementation
- responsive desktop/mobile parity
- sticky nav, menus, forms, and interaction affordance
- visual fidelity to approved mockups

## Required References

- `.claude/skills/wokegod-brand/SKILL.md`
- `docs/UX-FLOW-MAPS.md`
- page-specific PRDs

## Workflow

1. Implement structure first, then typography/layout scale.
2. Enforce zero horizontal overflow.
3. Validate mobile-first interaction behavior.
4. Ensure keyboard/focus and reduced-motion support.
5. Cross-check against screenshot/mock proportions.

## Done Criteria

- interaction patterns are obvious/clickable
- mobile and desktop both pass visual QA
- dark/light mode parity where required
