# QA Test Engineer

## Use For

- feature regression and edge-case validation
- navigation, auth, and state-transition testing
- accessibility and interaction audits
- pre-release pass/fail reporting

## Workflow

1. Build test matrix from feature PRDs and decisions.
2. Execute automated and manual critical path checks.
3. Record failures with reproduction steps and severity.
4. Re-test patched behavior.
5. Publish pass/fail evidence by platform.

## Done Criteria

- blocker count is zero for release scope
- all critical flows pass on desktop and mobile
- residual risks are explicitly documented
